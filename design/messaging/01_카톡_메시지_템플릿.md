# Q. D+0 카카오톡 메시지 템플릿 설계

> **작성 목적:** 다이어트 명함 앱의 핵심 바이럴 루프 — Scanner가 명함 스캔 후 수신자(Cardholder)에게 발송하는 카카오톡 비포&애프터 메시지의 표준 템플릿
>
> **작성일:** 2026-06-07
> **버전:** v1.0

---

## 0. 설계 원칙 (반드시 지킬 것)

| 원칙 | 이유 |
|---|---|
| **광고 톤 금지** | 정보통신망법 회피 — 영리 광고성 메시지가 아닌 P2P 인사로 |
| **수신자 이름 명시** | "당신의 명함" 보다 "{이름}님의 명함" 이 개인화 |
| **발신자 = 지인 본인** | "큐닷에서 보냄" 이 아니라 "박철수가 보냄" — 신뢰 전이 |
| **링크는 1개만** | 인지 부하 최소화. 가입 LP 단 하나 |
| **CTA는 명령형 X, 초대형 O** | "지금 가입하세요" → "같이 써볼까요?" |
| **이미지 우선** | 텍스트는 보조. 비포&애프터 이미지가 메시지의 본체 |

---

## 1. 카카오톡 메시지 카드 구조

```
┌──────────────────────────────────────┐
│  [발신자: 박철수]                    │
├──────────────────────────────────────┤
│                                      │
│   [비포애프터 이미지 카드 (large)]   │
│   ← 왼쪽: 기존 명함                  │
│   → 오른쪽: 8줄 → 4줄 다이어트본     │
│                                      │
├──────────────────────────────────────┤
│  본문 (텍스트 80자 내)               │
│                                      │
│  [버튼: 내 명함도 받기]              │
└──────────────────────────────────────┘
```

**기술 스택:** Kakao Talk Sharing API · Feed 템플릿 또는 Custom 템플릿

## 2. 메시지 본문 — 5종 A/B 후보

### 후보 A — 일반·캐주얼 (디폴트 추천)

> {Scanner}님이 {Cardholder}님의 명함을 가볍게 만들어 보냈어요.
> **8줄짜리 명함이 4줄로.** 010 빼도 메일이 도착해요.
>
> **[내 명함도 받기]**

**특징:** 가장 무난·친근. WOW 포인트(8→4)를 한 줄로 압축.
**카톡 미리보기 한 줄:** "{Cardholder}님의 명함이 가벼워졌어요"

---

### 후보 B — 비즈니스·신뢰

> {Cardholder}님, {Scanner}입니다.
> 전에 받은 명함을 디지털로 정리하면서 {Cardholder}님 것도 함께 만들었어요. 한번 보세요.
>
> **[디지털 명함 받기]**

**특징:** B2B·전문직 친화. 격식 톤. "함께 만들었다" 가 자연스러운 명분.
**카톡 미리보기 한 줄:** "{Scanner}님이 디지털 명함을 보냈어요"

---

### 후보 C — 임팩트·궁금증 유발

> 010 안 적어도 메일이 도착하는 명함, 들어보셨어요?
> {Scanner}가 먼저 {Cardholder}님 명함으로 만들어봤어요. 👇
>
> **[내 명함 확인하기]**

**특징:** 호기심 자극. 클릭률 가장 높을 후보. 다만 친밀도 낮은 관계엔 다소 가벼움.
**카톡 미리보기 한 줄:** "010 빼도 메일이 와요"

---

### 후보 D — 따뜻함·정중함

> {Cardholder}님 명함을 받고서,
> 이름이 곧 주소가 되는 서비스에 함께 등록해두었어요.
> {Cardholder}님 디지털 명함, 한번 확인해주세요.
>
> **[내 명함 보기]**

**특징:** 가장 정중·따뜻. 어른·시니어 대상에 적합. 신뢰도 최상.
**카톡 미리보기 한 줄:** "{Scanner}님이 디지털 명함을 보냈어요"

---

### 후보 E — 짧고 강력 (실험형)

> {Cardholder}님 명함, 더 가볍게 만들어봤어요.
>
> **[확인하기]**

**특징:** 미니멀. 텍스트 의존 최소. 이미지가 모든 걸 말함. 가장 위험·가장 강력.
**카톡 미리보기 한 줄:** "명함이 가벼워졌어요"

## 3. 동적 변수 처리

| 변수 | 값 예시 | 비고 |
|---|---|---|
| `{Scanner}` | 박철수 | Scanner의 카카오 표시명 |
| `{Cardholder}` | 김한국 | OCR 추출 이름 (성+이름) |
| `{Cardholder_Company}` | 큐닷주식회사 | OCR 추출 회사명 (옵션) |
| `{Image_BeforeAfter}` | URL | 자동 생성된 비포애프터 PNG |
| `{Signup_URL}` | https://q.co.kr/i/{Token} | 1회용 가입 토큰 |

**Signup 토큰 구조:**
`{Token}` = `{Scanner_Q-ID}` × `{Cardholder_Phone_Hash}` × `{Timestamp}`
→ Scanner 추천 보상 자동 귀속 + 1회용 + 만료 24h

## 4. A/B 테스트 권장 운영안

**1단계 (D+0~D+30, 창립회원 1만 명 시점):**
- 후보 A (디폴트) vs 후보 C (호기심) 양분 50:50
- 측정: 카톡 열람률, 클릭률, 가입 전환율, 24h 유지율

**2단계 (D+30~D+60):**
- 우승안 + 후보 B (비즈니스)·후보 D (따뜻함)을 **수신자 컨텍스트별** 자동 배정
  - Cardholder 직급 임원·대표 → 후보 B
  - Cardholder 시니어 (50대+) → 후보 D
  - 그 외 → 우승안

**3단계 (D+60+):**
- AI 자동 매칭 (Scanner-Cardholder 관계, 명함 업종, 시간대, 요일)

## 5. 스팸 방지 게이팅 (정보통신망법 컴플라이언스)

| 게이팅 | 규칙 | 이유 |
|---|---|---|
| **수신자 1회 거절 시** | 동일 Scanner의 재발송 영구 차단 | 정보통신망법 §50 |
| **1일 발송 상한** | Scanner 1인당 1일 30건 | 봇 어뷰징 방지 |
| **30분 쿨다운** | 동일 Cardholder에 중복 발송 30분 차단 | 실수 방지 |
| **신고 누적 3회** | Scanner 계정 일시 정지 + 큐닷 심사 | 진성 사용자 보호 |
| **링크 호스트 통일** | 항상 q.co.kr 서브패스만 사용 | 피싱 의심 회피 |

## 6. 비포애프터 이미지 자동 생성 사양

| 항목 | 사양 |
|---|---|
| 크기 | 800 × 1080px (카카오 권장) |
| 형식 | PNG, 95% 압축 |
| 좌측 (Before) | 원본 명함 OCR 결과 — 모든 정보 8줄 노출 |
| 우측 (After) | **회사 / 직함 / 이름 / [QR + Q메일]** 4줄 |
| 중앙 화살표 | `→ Q.` (오렌지) |
| 워터마크 | 우측 하단 `q.co.kr` (반투명) |
| Cardholder 이름 강조 | After 측에서 14pt 굵게 + 오렌지 |

## 7. 다음 화면 연결 (Signup LP)

카톡 클릭 → 모바일 LP로 즉시 이동.
LP는 별도 문서 (`02_모바일_LP.html`) 참조.

**LP 첫 화면 90% 일치 원칙:** 카톡 메시지에서 본 비포애프터 이미지가 LP 첫 화면에 그대로 등장해야 함. **인지 일관성** = 전환율의 최대 변수.
