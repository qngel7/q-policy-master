## 2026-07-13

### 기간
- 2026-07-12 ~ 2026-07-13 연속 작업
- 대상: 010.Q Q-메일 신청서, name.Q 유입 흐름, 모바일 입력 UX

### 작업 배경
- 010.Q에서는 휴대폰번호메일을 신청하고, name.Q에서는 이름메일 선점 가능 여부를 먼저 확인한 뒤 발급 버튼을 통해 010.Q 신청서로 이동한다.
- name.Q 유입 시 이름과 영문이름은 신청서에 자동 입력되므로, 사용자가 추가로 입력할 핵심 정보는 휴대폰번호와 실제 수신용 연결 이메일이다.
- 기존 하단 휴대번호 입력을 상단 `발급될 Q-메일`에 통합해 신청서를 단순화하고, 5060 사용자가 입력 위치를 쉽게 찾도록 시인성을 높이는 것이 목표였다.

### 만든 것
- 신청서 정보 구조 정리
  - `발급될 Q-메일`을 신청서 첫 항목으로 이동
  - 발급 메일 주소를 중앙 정렬
  - `가입 플랜`을 `플랜선택`, `신청인 이름`을 `이름`, `연결할 이메일`을 `연결 이메일`로 축약
- 이름 입력 UX 개선
  - 한글 이름과 영문이름을 한 줄 2칸으로 배치
  - 칸 비율을 3:7로 적용
  - 영문 라벨을 `영문이름(여권과 같게)`로 정리
  - OK-Q/Founder 이름메일 선점 가능 시 녹색 `@Q.co.kr` 안내가 입력칸 아래에 표시되도록 유지
- 연결 이메일 UX 개선
  - 아이디/도메인 분리 입력을 하나의 전체 이메일 입력창으로 변경
  - 브라우저가 완성한 전체 이메일도 그대로 허용
  - 네이버, 구글, 다음, 한메일, 아웃룩, 애플, 큐닷 버튼으로 도메인을 빠르게 완성
  - 이메일 지우기 버튼과 선택된 제공자 상태 표시 추가
- name.Q 연동 신청 흐름 정리
  - name.Q에서 선택한 이름을 한글·영문 이름칸에 자동 입력
  - 휴대번호 입력을 별도 하단 항목에서 상단 `발급될 Q-메일` 영역으로 통합
  - name.Q 유입 시 `010`을 기본 표시하고 커서를 `010` 뒤에 배치
  - 신청서 내부에는 휴대번호 입력 요소가 하나만 존재하도록 유지
- 발급 번호메일 표시 개선
  - 하이픈 없이 `01022223333@Q.co.kr` 형식으로 표시
  - `010`은 흰색, 국번 4자리는 오렌지, 마지막 4자리는 골드로 구분
  - 번호와 `@` 사이의 패딩·마진 및 글꼴 여백 제거
  - 전체 번호메일을 한 줄 중앙 정렬
- 5060 입력 시인성 개선
  - 이름·영문이름·연결 이메일 입력창에 어두운 배경과 블루 테두리 적용
  - 포커스 시 배경, 테두리, 외곽 강조
  - 모바일 이름·연결 이메일 행의 터치 높이와 상하 여백 확대

### 과정 메모
- 처음에는 name.Q의 이름메일과 휴대폰번호메일을 신청서 상단에 두 줄로 표시하는 것으로 잘못 이해했다.
- 실제 로직은 name.Q 페이지에서 이름메일 선점 확인을 마친 뒤 신청서로 이동하는 것이므로, 추가했던 이름메일 표시행을 제거하고 휴대번호 입력 위치만 상단으로 옮겼다.
- 첫 색상 구현은 단일 입력창에 퍼센트 그라데이션을 적용했다. 실기기에서 글꼴의 실제 숫자 폭과 그라데이션 경계가 달라 오렌지색이 인접 숫자 획을 침범하는 현상이 확인됐다.
- 최종적으로 `010`, 국번, 뒷번호를 독립된 표시 요소로 분리하고 실제 입력창은 투명 입력 레이어로 유지했다. 이 방식으로 색상 경계가 숫자 중간을 통과하지 않게 했다.
- 프로그램이 자동으로 채우는 번호와 사용자가 직접 입력하는 번호 모두 분리 표시 요소에 동기화되도록 보강했다.
- CSS/JS 캐시로 이전 화면이 남지 않도록 정적 자산 버전 문자열을 갱신했다.

### 별도 점검 사항
- Q-ID는 앞의 3자리 이니셜이 중복될 수 있지만 뒤 8자리 숫자 일련번호는 전체 회원에서 절대 중복되면 안 된다는 원칙을 재확인했다.
- 현재처럼 Q-ID 전체 문자열만 UNIQUE인 구조는 숫자 일련번호 중복을 막지 못하므로, 숫자 시퀀스 전역 유일성 제약과 정식 등록 전 데이터 정리가 필요하다.
- 향후 커스텀메일과 서브도메인 사용권의 등록·양도·경매 관리를 위한 별도 자산 관리 구조가 필요하며, 현재 가입 DB와 분리 설계하는 방향을 검토한다.
