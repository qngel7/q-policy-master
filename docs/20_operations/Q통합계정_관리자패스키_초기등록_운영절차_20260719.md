# Q. 통합계정 관리자 패스키 초기등록·복구 절차

> 승인일: 2026-07-19  
> 대상: 서대원, 서경석 관리자 계정  
> 원칙: 카카오·전화번호·비밀번호와 무관하게 로그인하되, 인증 없는 백도어는 만들지 않고 관리자 전용 패스키를 사용한다.

## 적용 결과

- 관리자 지정은 이름이나 전화번호가 아닌 변경되지 않는 `account_id`를 기준으로 한다.
- 관리자 계정은 일반 비밀번호, 카카오 로그인, 카카오 번호 계정복구로 로그인할 수 없다.
- 관리자 등록 절차에서 생성한 `is_admin_key=true` 패스키만 관리자 로그인에 사용할 수 있다.
- 관리자는 서로에게 10분짜리 일회용 등록·복구 링크를 발급할 수 있다.
- 각 관리자는 오프라인 복구코드 8개를 발급하여 별도 보관할 수 있다.
- 기존 일반 패스키는 최초 관리자 패스키 등록 완료 시 폐기된다.
- 마지막 관리자 패스키 한 개는 API와 DB 함수에서 원자적으로 삭제를 차단한다.
- 역할 지정, 링크 발급·사용, 패스키 등록·로그인·폐기, 복구코드 발급·사용을 감사로그에 기록한다.

## 1. 선행 배포

Supabase SQL Editor에서 다음 마이그레이션을 먼저 실행한다.

```text
services/010/db/add_passwordless_auth.sql
```

Cloudflare에는 일반 패스키 배포와 같은 설정을 사용한다.

```text
AUTH_PASSKEY_ENABLED=true
PASSKEY_RP_ID=q.co.kr
PASSKEY_ORIGIN=https://010.q.co.kr
PASSKEY_ALLOWED_ORIGINS=https://010.q.co.kr
```

## 2. 두 회원의 정확한 account_id 확인

이름은 검색에만 사용하고 결과의 Q-ID·전화번호를 사람이 대조한다. 동명이인을 확인하지 않은 채 관리자 역할을 지정하면 안 된다.

```sql
SELECT account_id, q_id, phone, kor_name, status
FROM registrations
WHERE kor_name IN ('서대원', '서경석')
ORDER BY kor_name, created_at DESC;
```

## 3. 관리자 역할 지정

아래 자리에는 위 조회에서 확인한 실제 UUID를 입력한다.

```sql
SELECT q_set_admin_account('<서대원_ACCOUNT_ID>'::uuid, 'owner');
SELECT q_set_admin_account('<서경석_ACCOUNT_ID>'::uuid, 'admin');
```

역할 지정 시 기존 로그인 세션은 자동 무효화된다.

## 4. 서대원 최초 관리자 패스키 등록

SQL Editor에서 최초 등록 링크를 한 번 발급한다.

```sql
SELECT
  'https://010.q.co.kr/admin/enroll?token=' ||
  q_create_admin_enrollment(
    '<서대원_ACCOUNT_ID>'::uuid,
    NULL,
    'bootstrap',
    15
  ) AS enrollment_url;
```

1. 반환된 URL을 서대원 본인 기기에서 연다.
2. `이 기기에 관리자 패스키 등록`을 누른다.
3. 지문·얼굴·화면 잠금 확인을 완료한다.
4. `/admin/security` 화면이 열리는지 확인한다.

## 5. 서경석 관리자 패스키 등록

1. 서대원이 관리자 패스키로 로그인한다.
2. `/admin/security`에서 서경석 Q-ID 또는 전화번호를 입력한다.
3. `다른 관리자 계정 복구`를 선택하고 일회용 링크를 발급한다.
4. 링크를 서경석 본인 기기로 안전하게 전달한다.
5. 서경석이 링크를 열고 자신의 지문·얼굴·화면 잠금으로 패스키를 등록한다.

관리자가 없는 초기 상태에서만 SQL Editor의 `bootstrap` 발급을 사용한다. 정상 운영에서는 한 관리자가 다른 관리자의 복구 링크를 발급한다.

## 6. 필수 후속 설정

두 관리자 모두 다음을 완료한다.

1. 휴대전화와 PC 등 관리자 패스키를 최소 2개 등록한다.
2. `/admin/security`에서 오프라인 복구코드 8개를 발급한다.
3. 복구코드는 온라인 메모·카카오톡에 저장하지 않고 인쇄물 또는 별도 오프라인 저장소에 보관한다.
4. 이전 기기를 분실했다면 새 기기 등록 후 분실 기기의 패스키를 폐기한다.

## 7. 이후 로그인

```text
Q. 통합계정 로그인
→ 지문·얼굴·화면 잠금으로 로그인
→ 기기에서 서대원 또는 서경석 관리자 패스키 선택
```

카카오 등록번호나 회원 전화번호가 달라도 관리자 패스키 로그인에는 영향이 없다.

## 8. 복구 우선순위

1. 같은 관리자의 두 번째 등록 기기
2. 다른 관리자가 발급한 10분짜리 복구 링크
3. 본인이 오프라인으로 보관한 1회용 복구코드
4. 모두 없을 때만 Supabase SQL Editor에서 새 `bootstrap` 링크 발급

실회원 관리자계정은 시연용 공용계정으로 사용하지 않는다. 여러 사람 앞에서의 가입 시연에는 별도 `demo_operator` 계정과 가상 회원데이터를 사용한다.
