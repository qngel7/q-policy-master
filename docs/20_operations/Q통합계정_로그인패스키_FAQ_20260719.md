# Q. 통합계정 로그인·패스키 FAQ

> 작성일: 2026-07-19  
> 상태: 프리런칭 베타 운영 기준  
> 공개 페이지: `https://010.q.co.kr/faq.html`

## 1. 기본 원칙

### 패스키란 무엇인가요?

비밀번호 대신 휴대폰·컴퓨터의 지문, 얼굴, 화면잠금으로 로그인하는 방식이다. Q. 서버에는 로그인 검증용 공개키만 저장하며 지문·얼굴 원본은 기기 밖으로 전송하거나 저장하지 않는다.

### 안드로이드와 아이폰에서 모두 작동하나요?

작동 원칙은 같다. 안드로이드는 Google 비밀번호 관리자나 Samsung Pass 등에, 아이폰은 iCloud 키체인 등에 패스키가 저장될 수 있다. 메뉴 이름과 인증창 모양은 기기·OS 버전에 따라 다르다.

### 어떤 브라우저를 사용해야 하나요?

안드로이드는 Chrome 또는 삼성 인터넷, 아이폰은 Safari를 권장한다. 카카오톡·문자 앱 내부 브라우저는 패스키 제공자 선택과 쿠키가 시스템 브라우저와 분리될 수 있으므로 링크를 복사해 시스템 브라우저에서 연다.

## 2. 패스키 저장소와 여러 계정

### Google 비밀번호 관리자와 Samsung Pass에 모두 패스키가 보입니다.

한 휴대폰에 패스키 제공자가 여러 개 활성화되면 같은 Q-ID가 여러 저장소에 보일 수 있다. 로그인할 때 `패스키 더보기`에서 실제 등록에 사용한 제공자를 선택한다. 정상 로그인을 확인하기 전에는 같은 이름의 패스키를 무작정 삭제하지 않는다.

### 한 휴대폰에 여러 Q. 계정을 등록할 수 있나요?

가능하다. 패스키 선택창에서 로그인할 Q-ID를 선택한다. 다만 브라우저 한 개에는 한 계정 세션만 유지되므로 동시에 관리하려면 Chrome과 삼성 인터넷처럼 브라우저를 나누는 방법을 사용할 수 있다.

### 제공자를 하나로 통일해야 하나요?

필수는 아니지만 혼동을 줄이려면 한 기기에서는 기본 패스키 제공자를 하나로 정하는 편이 좋다. 제공자 변경이나 중복 정리는 정상 패스키 로그인을 확인하고 복구수단을 준비한 뒤 진행한다.

## 3. 일반회원 로그인과 복구

### 평상시에는 어떻게 로그인하나요?

`Q. 통합계정 로그인`에서 `지문·얼굴·화면 잠금으로 로그인`을 누르고 본인의 Q-ID 패스키를 선택한다.

### 패스키가 없거나 새 휴대폰이면 어떻게 하나요?

일반회원은 카카오 계정 확인으로 로그인하거나 계정을 복구한 뒤 새 기기에 패스키를 등록한다.

### 비밀번호를 모르는데 계정을 복구할 수 있나요?

가입한 휴대전화번호를 입력하고 카카오 인증을 진행한다. 가입번호와 카카오가 제공한 전화번호가 일치하면 비밀번호 없이 복구된다.

### 가입번호와 카카오 등록번호가 다르면 어떻게 되나요?

현재 베타에서는 자동복구되지 않는다. 관리자 확인 대상으로 남기며, 향후 SMS/PASS 복구수단이 도입되면 별도 절차를 제공한다.

## 4. 관리자 로그인과 복구

### 관리자는 카카오나 비밀번호로 로그인할 수 있나요?

관리자 권한 로그인은 관리자 등록 절차에서 만든 전용 패스키만 인정한다. 일반 패스키·카카오·비밀번호로 관리자 권한을 열지 않는다.

### 관리자 휴대폰을 잃거나 패스키가 작동하지 않으면 어떻게 하나요?

다른 관리자가 자신의 관리자 패스키로 로그인한 뒤 대상 관리자의 Q-ID 또는 전화번호로 10분짜리 일회용 복구 링크를 발급한다. 대상 관리자는 시스템 브라우저에서 링크를 열어 새 관리자 패스키를 등록한다.

### 다른 관리자도 사용할 수 없으면 어떻게 하나요?

본인이 미리 오프라인으로 보관한 일회용 복구코드를 사용한다. 복구코드는 다른 사람에게 전달하지 않으며 새로 발급하면 이전 미사용 코드는 모두 폐기된다.

### 한 휴대폰에 관리자 두 명의 패스키를 둘 수 있나요?

기술적으로 가능하지만 같은 휴대폰의 화면잠금으로 두 관리자 신원을 모두 사용할 수 있고 감사기록도 선택한 계정 이름으로 남는다. 비상·백업용으로만 사용하고 평상시에는 각자 계정을 사용하는 것을 권장한다.

## 5. 자주 보이는 오류

### `An unknown error occurred while talking to the credential manager.`

카카오톡 내부 브라우저이거나 패스키 제공자가 여러 개인 경우 발생할 수 있다. 페이지를 닫고 시스템 브라우저를 직접 연 뒤 `패스키 더보기`에서 Google 비밀번호 관리자·Samsung Pass 등 올바른 제공자를 선택한다.

### `NOT_FOUND`

서버에 없는 패스키, 이미 폐기된 패스키, 다른 제공자에 남은 과거 패스키를 선택했거나 로그인 요청이 만료된 경우다. 새 로그인 페이지에서 다시 시도하고 올바른 제공자를 선택한다. 관리자라면 정상인 다른 관리자가 복구 링크를 발급한다.

### `이미 등록된 패스키` 또는 `Q. 관리자에서 사용할 패스키가 이미 있습니다.`

해당 기기·제공자에 서버가 알고 있는 패스키가 이미 있다는 뜻이다. 재등록을 반복하지 말고 로그인 화면에서 그 제공자를 선택한다. 그래도 로그인되지 않으면 정상 복구수단을 확보한 뒤 중복 패스키를 정리한다.

### 패스키를 삭제해도 되나요?

정상 로그인을 확인하기 전에는 삭제하지 않는다. 특히 관리자는 다른 관리자 복구 또는 오프라인 복구코드를 준비하고, 새 패스키로 실제 로그인이 성공한 뒤 오래된 기기만 등록 해제한다. 마지막 관리자 패스키는 삭제할 수 없다.

## 6. 휴대폰 교체·분실

- 새 휴대폰에서 동기화된 패스키가 보이면 본인 계정을 선택해 로그인한다.
- 동기화되지 않았다면 일반회원은 카카오 번호일치 복구, 관리자는 다른 관리자 링크 또는 오프라인 복구코드를 사용한다.
- 새 패스키 로그인을 확인한 뒤 분실 기기의 서버 등록을 해제한다.
- 패스키 제공자의 클라우드 동기화만 유일한 복구수단으로 믿지 않는다.
