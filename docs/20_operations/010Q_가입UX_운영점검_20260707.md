# 010.Q 가입 UX/운영전략 점검 — 2026-07-07

## 점검 결론

010.Q는 이제 "번호메일 발급"보다 "명함 기반 신뢰 가입"을 앞세우는 흐름으로 운영해야 한다. ForwardEmail 실수신이 확인된 상태이므로 다음 병목은 메일 인프라가 아니라 가입자가 자기 정보와 받을 메일을 확신하는 UX다.

## 1. 실제 가입 UX

- 직접 유입자는 먼저 번호를 입력하고, 골드번호 여부를 확인한다.
- 무료 010-Q는 골드번호 보유자 우선 메시지로 희소성을 만든다.
- 일반번호 사용자는 가입 불가처럼 보이면 안 된다. 창립회원 초대, 전문가회원, 명함스캔 가입으로 이어준다.
- B가 QR/카톡 링크로 들어오면 즉시 인증부터 누르게 하지 않고, 이름/전화번호/받을 메일을 확인한 뒤 카카오 인증으로 이동한다.

성공 기준:
- 사용자가 발급될 Q메일과 실제 받을 메일을 구분한다.
- 무료/OK-Q/Founder의 가입 맥락을 이해한다.
- 카카오 인증 전 입력값을 한 번 확인한다.

## 2. 명함스캔 정확도와 누락 필드

명함스캔은 완전 자동발급 도구가 아니라, OCR 초안을 사람이 확인하는 도구로 설계해야 한다.

필수값:
- Q메일 발급/전달: 휴대전화번호 + 받을 메일
- 명함만 저장: 이름 또는 회사 등 일부 정보만 있어도 가능

운영 기준:
- 휴대전화번호가 없으면 Q메일 발급 불가, 직접 입력 유도
- 받을 메일이 없으면 ForwardEmail 연결 불가, 직접 입력 유도
- 둘 중 하나라도 없으면 "명함만 저장" 대안을 유지
- 비포 명함에는 실제 명함 이메일 또는 수정 입력 이메일을 보여준다

## 3. 창립회원 가입경로

Founder는 골드번호 판정과 별개다. 2026년 창립회원은 무작위 오픈가입보다 초대, 명함스캔, 현장 가입, 전문가 네트워크를 통해 받는다.

화면 원칙:
- Founder 선택 시 "2026 창립회원 전용"을 명시
- 결제/혜택 상세는 별도 Founder 페이지 또는 운영자 안내로 분리
- 가입 완료 후 V-ID 추가발급 예정 메시지는 유지하되, 일반 Q-ID와 혼동되지 않게 한다

## 4. 골드번호 일반회원 전략

무료회원 제한은 차단 장치가 아니라 이야기 장치다.

기본 제공:
- Q-ID
- V-ID-010, 즉 핸드폰번호메일

유료 추가 V-ID:
- V-ID-Gold
- V-ID-Fandom
- V-ID-Pro

추가 V-ID는 이니셜구독 월 $1과 별개로 월 $1 추가 구독을 요구한다. 단, V-ID 등급에 따라 별도 선택 가격 또는 프리미엄 가격을 둘 수 있다.

골드번호 판정 우선 패턴:
- AAAA, A000, 000A
- AA00, 00AA, ABAB, AABB
- ABCD, DCBA

점수 표시 초안:
- 100%: 5연속+010 같은 시그니처형. 예: 77777010
- 99%: 앞뒤 반복 블록. 예: 77777777
- 90% 이상: 끝 4자리 반복. 예: 임의 앞자리 + 7777
- 80% 이상: 연속, ABAB, AABB 등 강한 기억 패턴
- 50%: 일반 임의번호

운영 메시지:
- 골드번호: "무료 010-Q 우선 가입 대상"
- 일반번호: "창립회원 초대, 전문가회원, 명함스캔 가입으로 진행 가능"

## 5. 명함스캔 기반 초대/가입 흐름

권장 흐름:

1. A가 B 명함 스캔
2. OCR 결과 확인/수정
3. 휴대전화번호와 받을 메일 상태 표시
4. 전달 가능하면 temp 레코드 생성
5. QR/카톡으로 B에게 전달
6. B가 정보 확인/수정
7. 카카오 인증
8. active 승격, ForwardEmail alias 생성, 실제 수신 테스트

운영 테스트 세트:

| 케이스 | 기대 결과 |
|---|---|
| 전화번호+받을메일 있음 | 바로 전달 가능 |
| 전화번호 있음, 받을메일 없음 | 받을 메일 입력 요구 |
| 전화번호 없음, 받을메일 있음 | 휴대전화번호 입력 요구 |
| 둘 다 없음 | 명함만 저장 유도 |
| 이미 가입된 번호 | 전달 차단 |
| 골드번호 직접 유입 | 무료 010-Q 가입 메시지 |
| 일반번호 직접 유입 | 창립/전문가/명함스캔 대체 경로 안내 |
