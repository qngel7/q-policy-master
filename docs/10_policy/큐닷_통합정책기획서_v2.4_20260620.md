# §3 010Q · OKQ 메일 시스템 (v2.3 개정)

> **핵심 명제:** 번호가 곧 이메일, 이메일이 곧 번호 — 무료(010Q)와 유료(OKQ)가 명확히 분리된다.

## §3-A 010Q메일 (무료 · 기본 등급)

| 항목 | 내용 |
|---|---|
| **가입 경로** | 홈페이지 접속 (추천링크 또는 QR코드) |
| **가입 조건** | 휴대폰 본인인증 |
| **발급 성격** | **인증 즉시 발급** — 선점 예약이 아님, 실사용 주소 즉시 활성 |
| **가격** | 무료 · 영구 |
| **이메일 형식** | `01070008000@Q.co.kr` (010 포함 11자리) |
| **타깃** | 전 국민 (휴대폰 보유자) |
| **Q-vic 적립** | ✅ 활동(출석·설문·추천 등)으로 획득 가능 |
| **Q-vic 사용** | ❌ 이니셜 구독(OKQ 플랜) 가입 후 사용 가능 |

### 가입 흐름

```
추천링크·QR 접속
    ↓
홈페이지 신청서 작성 (번호 입력)
    ↓
휴대폰 본인인증 (카카오 OAuth 또는 SMS)
    ↓
010Q 메일 + Q-ID 즉시 발급 ← 선점·예약 없음, 실사용 주소 바로 활성
```

## §3-B OKQ메일 (유료 · 이니셜 구독 등급)

### OKQ 네이밍 의도 (v2.3 신설)

> **OKQ = "010, OK, Q"**
> - **010 붙여도·빼도 수신 OK** (7000-8000@Q.co.kr 과 010-7000-8000@Q.co.kr 모두 수신)
> - 명함에 번호 없이 이름만 쓰는 시대 — 이름@Q.co.kr 로 수신
> - **명함 사용자(직장인·전문가)** 특화 서비스

| 항목 | 내용 |
|---|---|
| **가입 경로** | 명함 스캔 (기본) 또는 010Q에서 구독 업그레이드 |
| **가격** | **월 ₩1,500 (= $1) · 이니셜 구독** |
| **이메일 형식** | `70008000@Q.co.kr` (010 제외 8자리) **[A안 확정]** |
| **발급 원칙** | 8자리(010 제외)로 발급. 010Q(11자리)와 시각적으로 구분 → OKQ 등급 명함에서 즉시 식별 |
| **수신 호환** | `01070008000`, `010-7000-8000`, `70008000`, `7000-8000`, `010.7000.8000` 등 모든 표기 수신 |
| **구독 만료 시** | 8자리 주소(`70008000@Q`) 수신 차단 / 010Q 주소(`01070008000@Q`)는 무료로 계속 유효 |
| **내이름메일** | `김한국@Q.co.kr` — 한글 이름 기반 자동 예약 (양도 가능) |
| **영문여권이름메일** | `KIMHANKOOK@Q.co.kr` — 여권 기준 대문자 추가 생성 (양도 가능) |
| **타깃** | 직장인·전문가·명함 보유자 |
| **Q-vic 사용** | ✅ 취득 및 자유 사용 |
| **Q-bit 사용** | ✅ 취득 및 Q-bit 전환 |

### 명함 없는 사용자의 OKQ 진입 경로 (v2.2 유지)

- 명함 미보유자는 다이어트 명함 앱이 AI로 임시 명함 자동 생성 → OCR 입력 명함으로 OKQ 가입 조건 충족
- 리멤버·토스명함 등 기존 명함 앱을 통해서도 가입 가능 (API 연동 협의 중)

# §7 회원 정책 (v2.4 개정)

## §7-0 회원아이디 발급 규칙 (v2.4 신설)

**기본 회원아이디(Q-ID)** — 전 회원 공통:
`이니셜 3자(영문 이름 단어별 첫 글자, 대소문자 무시 → 대문자) + 8자리 순번`, 하이픈 없음.
순번은 전 회원 공통 단일 시퀀스 `20000000`부터. 예: `ABC20000000`.

**V-ID(추가 ID)** — 동일 형식 `이니셜 3자 + 8자리 순번`:

| 대상 | 발급 방식 | 순번 구간 | 예시 |
|---|---|---|---|
| **창립회원(Founder)** | 자동 추가 발급 | `0`부터 (`0xxxxxxx`) | `ABC00000000` |
| 일반회원 | **선택 발급 — 현재 보류** | `20000000` 이상 & 유보 제외 | `ABC2xxxxxxx` |

**유보(reserved) 구간:**
- 기본 유보: `00000000 ~ 19999999` (창립 `0xxxxxxx` 포함, 일반 선택 불가) — 고정.
- 추가 유보: **경매유보순번** — 정책 결정 후 유보목록(`reserved_vid_numbers`)에 추가.

> 발급 시점: 가입 시 임시 기록(`q_id="TEMP:..."`, 번호 기반) → 카카오/관리자 인증으로 `status: temp→active` 전환 시 DB 트리거가 정식 발급. 창립회원은 이때 V-ID도 함께 자동 발급.

## §7-A Q-ID · V-ID · 자산 아키텍처 (v2.5 신설 — 헌법 제7조 구체화)

### 기본 구조

```
[신원]  Q-ID  ←  1인 1개, 절대 고유, 양도 불가 (실명 인증으로 고정)
              ↓
[식별]  V-ID  ←  Q-ID에 귀속, 다수 허용 (직업·역할·법인용)
              ↓
[자산]  메일주소 · 서브도메인 · 번호  ←  소유·양도·경매 가능
```

> **핵심 원칙:** Q-ID만이 신원이다. 그 외 모든 디지털 주소와 식별자는 자산이다.

### 자산 분류표

| 자산 | 유형 | 1인 보유 한도 | 양도 가능 | 비고 |
|---|---|---|---|---|
| 한글이름메일 | 이름메일 Slot A | 1개 | ✅ | OKQ 이상 자동 예약 |
| 영문여권이름메일 or 이니셜메일 | 이름메일 Slot B | 1개 (택 1) | ✅ | Founder 이상, §7-B |
| 번호메일 | 번호 자산 | 1계정 기본 + 추가 유료 | ✅ | §7-C |
| 서브도메인 | 도메인 자산 | 무제한 | ✅ | 연간 유지비 |
| 자연어·브랜드 주소 | 자유시장 자산 | 무제한 | ✅ | 이름 제한 없음 |

## §7-B 이름메일 2-슬롯 원칙 (v2.5 신설)

모든 구성원의 이름메일은 최대 **2슬롯**으로 제한된다.

| 슬롯 | 형식 | 발급 조건 | 변경 |
|---|---|---|---|
| **Slot A (고정)** | 한글이름@Q.co.kr | OKQ 이상, 자동 예약 | 이름 변경 시 1회 가능 |
| **Slot B (택 1)** | 영문여권이름@Q.co.kr **또는** 이니셜메일@Q.co.kr | Founder 이상 | 최초 선택 후 변경 불가 |

**Slot B 배타적 선택 규칙:**

- 영문여권이름 선택 → 이니셜메일 추가 발급 불가
- 이니셜메일 선택 → 영문여권이름 미발급
- 둘 중 하나만. 선택 이후 변경 불허.

**이니셜메일 형식 (Slot B 선택 시):**

- 형식: `성(영문).이름이니셜@Q.co.kr`
- 예: 서대원 → `Seo.DW@Q.co.kr` (Seo=서, D=대, W=원)

**충돌 시 cascade 순서:**

```
Seo.DW@Q (1순위)
  └─ 충돌 → SDW@Q (이니셜 3자 축약)
       └─ 충돌 → Seo.Daewon@Q (성+풀이름)
            └─ 충돌 → Seo.DW2@Q (번호 부여)
                 └─ 충돌 → 사용자 직접 지정
```

# §10 Q-vic · Q-bit 경제 시스템 (v2.2 유지 + 가격 반영)

## §10-1 이니셜 구독과 Q-vic 관계 원칙

- **1 Q-vic = ₩1** (유지)
- **이니셜 구독료(₩1,500/월)는 Q-vic으로 결제 불가** — 현금·카드만 허용

- 이니셜 구독 미납 시 Q-vic 잔액은 동결 (적립은 계속)
