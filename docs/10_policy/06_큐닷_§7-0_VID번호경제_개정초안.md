# 큐닷 정책 §7-0 개정 초안 — V-ID 번호경제 (DRAFT)

**기준 버전:** v2.4 (2026-06-20)
**초안 작성:** 2026-06-22 (Cowork / Claude assist)
**상태:** **초안 — 대표(서대원) 확정 · 법무 자문 전. 코드/SQL 미반영.**
**범위:** v2.4 §7-0(회원아이디 발급 규칙) 전면 대체. 승인 시 v2.5 §7-0로 편입.

---

## 0. 이 문서의 위치

v2.4 §7-0은 "기본순번 `20000000~` 시작 / `00000000~19999999` 유보 / 창립 V-ID `0~`"였다.
본 초안은 이를 **번호=실휴대번호 매칭 경제(V-ID)** 모델로 재정의한다. 본 문서는 **기획 초안**이며, DB 시퀀스·트리거 변경은 **§7 별도 승인 후** 진행한다(영향 큼 — 라이브 010과 동일 Supabase).

---

## 1. 번호 구간 재정의

| 순번 구간 | 용도(초안) | v2.4 | 비고 |
|---|---|---|---|
| `00000000 ~ 09999999` | **기본 Q-ID 순번** (서경석 `00000000`부터) | 유보였음 | 첫 1천만 회원 기본 ID |
| `10000000 ~ 19999999` | **유보 — 전국대표번호(1588 등) Q Business 티어** | 유보(미지정) | 대표번호 비즈니스용으로 활용 |
| `20000000 ~ 99999999` | **실번호 매칭 구간** — 평시 V-ID(번호매칭), 1천만 초과 시 기본순번 자동 연장 | 기본순번 시작구간이었음 | 실휴대번호와 동일 수치 공간 |

**근거(웹검증 완료):** 국내 010 국번은 `2000`부터 실배정되고, `010-1xxx`(전국대표번호 충돌 방지)·`010-0xxx`는 미사용 → **`00000000~19999999`에는 실제 휴대전화 번호가 없다(충돌-프리).** `20000000~99999999`가 실번호 구간이다.

---

## 2. 기본 Q-ID (base serial)

- **형식:** 이니셜 3자(영문 이름 단어별 첫 글자, 대소문자 무시 → 대문자) + 8자리 순번, **하이픈 없음**. 예 `SKS00000000`.
- **순번:** 전 회원 공통 단일 시퀀스. **`00000000`(서경석)부터** 오름차순.
- **연장:** 누적 1천만 도달 시 `10000000~19999999`(유보)를 **건너뛰고 `20000000`부터** 자동 연장.
- 기본 Q-ID는 가입자의 실번호와 무관한 **순번 신원**이다(실번호 매칭은 V-ID가 담당).

---

## 3. V-ID (번호매칭) — 의미 재정의

> v2.4의 "창립 보너스 `0~`"에서 → **"본인 휴대번호와 매칭하는 베니티 ID"**로 재정의.

| 항목 | 내용 |
|---|---|
| **구간** | `20000000 ~ 99999999` (실번호 구간) |
| **형식** | 이니셜 3자 + 8자리(=본인 휴대번호 뒷 8자리), 하이픈 없음 |
| **자격** | **구독회원(OKQ 이상) 한정** |
| **본인성** | 본인인증으로 **자기 번호만** 매칭 가능 — 타인 번호 매칭 불가(사칭·사생활 방지) |
| **소진 규칙** | 기본순번이 성장하여 해당 번호 값에 도달·배정되면, 그 번호의 V-ID는 **확보 불가** |
| **성격** | 소유권 아님 — **플랫폼 사용권 라이선스** |

⚠️ **결정 필요 (v2.4와 충돌):** 기본순번이 `0~`로 시작하면서, 초창기 낮은 번호(서경석=0 등)가 **사실상 창립 프레스티지**가 된다. v2.4의 "Founder 전용 V-ID `0~` 자동 발급"과 의미가 겹친다.
→ 택1 필요: **(a)** Founder 별도 ID 폐지(낮은 기본순번으로 프레스티지 대체) / **(b)** Founder에게 별도 표식 유지(예: 번호매칭 V-ID 우선권·수수료 면제 등). **권장: (a) 단순화.**

## 4. 사용권 2단 구조 (디지털 부동산)

큐닷 헌법의 "서브도메인 임대 = 디지털 부동산, 사용권" 프레임을 번호 신원에 적용.

| 권리 | 정의 |
|---|---|
| **1차 사용권** | 최초 수분양자의 **자가 사용권** (주인이 직접 거주) |
| **2차 사용권** | **임대권** — 수분양자가 타인에게 빌려주어 수익 (세를 놓음) |

**분쟁(법적 우선권리자 = 상표권자 등) 처리:**

1. **법 우선** — 우선권리자의 정당한 권리는 인정한다.
2. **최선 시나리오(자발적 합의):** 수분양자가 1차(임대인)로 남고, 권리자가 **2차(임차)로 사용료 납부** → **Q·수분양자 정산 분배**.
3. **상표권 문제 발생 시:** 수분양자는 해당 번호를 **제3자에게 2차 임대 불가**.
4. ⚠️ **한계 명시:** "권리자가 임대료를 내고 쓴다"는 **권리자가 자발 합의할 때만** 성립한다. 법적으로는 권리자가 **무상 회수 + 선점자에 손해배상**을 주장할 수 있으므로, 본 모델은 **합의·정산 시장 레이어**이지 권리자에게 강제할 수 있는 구조가 아니다.

## 5. 유보·분쟁 규정 (Sunrise)

- **선제 유보 리스트(Sunrise):** 저명상표·공공기관·위험/욕설 패턴을 사전 차단(`reserved_*` 목록).
- **일반(비저명) 상표:** 상표권은 지정상품류 한정 → `.Q` 신규 네임스페이스에서 **공존 가능**(삼성통닭·삼성부동산 공존 원리).
- **저명상표(삼성·애플 등):** 부정경쟁방지법·희석화 법리로 **분야를 넘어 보호** → **유보 대상**. 상업적 사용 시 혼동·명성손상 리스크.
- **분쟁해결규정(UDRP식):** 선의(good-faith) 선점만 인정, **악의적 선점 무효**, 법적 우선권리자 우선 + **선의 수분양자 보상/정산을 Q가 중개**. Q는 **중립 정산·라이선스 발행자**로 포지셔닝(조장 책임 회피).

---

## 6. 자격·게이트

- **V-ID 번호매칭 / 1차·2차 거래 = 구독회원(OKQ 이상) 한정.**
- **대표번호 Q Business 티어(10M~20M) = 검증된 운영자 한정** — 통신사 가입증명·사업자등록 대조. (B2B 상위 티어, 별도 가격)
- 약관에 **"소유권이 아닌 플랫폼 사용권 라이선스"** 명시.
