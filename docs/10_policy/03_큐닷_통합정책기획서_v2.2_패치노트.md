# §5 다이어트 명함 앱 (전면 개정)

> **비전:** "당신의 명함이 가벼워집니다" — 그리고 **그 가벼움이 다음 가입자를 부릅니다.**

## §5-1 핵심 컨셉 (유지)

8줄 → 4줄. 회사명 · 직함 · 이름 · [QR + Q메일]. QR은 Q-ID 프로필 연결, 휴대번호 변경 시에도 컨택 지속.

## §5-2 **새 D+0 바이럴 루프** (핵심 신설)

기존 v2.1의 "사용자 다운로드 → 가입" 모델을 폐기. 새 모델은 **수신자 본인 명함이 카톡으로 도착하는 방식**.

### D+0 흐름 (수신자 = 김한국 시점)

```
1. Scanner(박철수)가 김한국 명함을 받아 스캔
   ↓
2. 박철수 앱이 OCR → 비포애프터 자동 렌더링
   ↓
3. 박철수 카카오톡으로 김한국에게 "당신의 디지털 명함" 발송
   ↓
4. 김한국 카톡 수신 → 본인 명함 변신본 확인 (WOW)
   ↓
5. [내 명함 받기] 클릭 → 모바일 LP 진입
   ↓
6. 카카오 인증 1탭 → 010Q메일 즉시 발급
   ↓
   ✅ 가입 완료. 다운로드·비밀번호·양식 없음.
```

### 왜 이게 역대 가장 강력한 바이럴인가

| 서비스 | 수신자가 받는 것 | 동기 강도 |
|---|---|---|
| Dropbox | 텍스트 초대장 | 약함 |
| Hotmail | 메일 푸터 한 줄 | 매우 약함 |
| **Q.** | **본인의 변신된 명함** | **압도적** |

자기 이름·자기 정보가 담긴 결과물은 거부 불가능. **광고가 곧 결과물, 결과물이 곧 광고.**

## §5-3 카카오톡 메시지 표준 (요약)

상세 템플릿은 별도 문서 `01_카톡_메시지_템플릿.md` 참조.

### 디폴트 메시지 (후보 A)

> {Scanner}님이 {Cardholder}님의 명함을 가볍게 만들어 보냈어요.
> **8줄짜리 명함이 4줄로.** 010 빼도 메일이 도착해요.
>
> **[내 명함도 받기]**

### 컴플라이언스 핵심

- 정보통신망법 §50 회피: 발송 주체 = Scanner 개인 (P2P), 큐닷 본사 X
- 1일 발송 상한: Scanner 1인당 30건/일
- 수신 거절 시: 동일 Scanner 재발송 영구 차단
- 신고 누적 3회: Scanner 계정 일시 정지

상세 디자인은 `02_모바일_LP.html` 참조.

## §6-2 **이니셜 구독 (Initial) — 신설** (C3 핵심)

### 원칙

> Q-vic을 아무리 많이 쌓았어도, **이니셜 구독이 활성 상태가 아니면 사용 불가.**

"이니셜(Initial)" = ① 시작·진입권 ② Q-ID 이니셜3자 정체성 ③ 첫 글자.

### 활성 vs 비활성 매트릭스

| 상태 | Q-vic 적립 | Q-vic 사용 | Q-bit 사용 | 도메인 청약 |
|---|---|---|---|---|
| **이니셜 활성** | ✅ | ✅ | ✅ | ✅ |
| **이니셜 비활성** | ✅ | ❌ (동결) | ❌ (동결) | ❌ |
| **창립회원** | ✅ | ✅ (평생) | ✅ (평생) | ✅ (우선권) |

### 동결·해지·재활성화 규칙

| 시나리오 | 처리 |
|---|---|
| 구독 해지 | Q-vic·Q-bit 잔액 **동결** (삭제 X) |
| 재구독 | 동결 잔액 **100% 복구** |
| 12개월 연속 비활성 | 좀비 회원 정리 — 매월 ₩1,000 Q-vic 자동 소진 시작 |

| 모두 0 | 계정 동결 (재활동 신청 시 복구) |

| 활동 | 일일 상한 | 월 최대 상쇄 | 어뷰징 방지 |

새 D+0 모델 기반 보수적 계산:

```
1 OKQ회원 × 5스캔/월 × 50% 가입 전환 = 2.5 신규 010Q회원/월
누적 3개월 010Q 신규 = 7.5명
7.5명 × 20% OKQ 전환 (이니셜 잠금 압력) = 1.5 신규 OKQ회원/3개월

→ 바이럴 계수 K = 1.5 (90일 주기) — 자연 복리 성장 임계점 초과
```
