# 2026 창립회원 모집계획 및 가입경로 시뮬레이션

작성일: 2026-07-07

## 1. 전략 방향

2026년 010.Q의 회원 모집은 무작위 대량 가입보다 회원의 밀도, 신뢰도, 이야기성을 우선한다.

일반 무료회원은 골드번호 보유자로 한정하여 희소성과 화제성을 만든다. 다만 서비스 확산을 막는 폐쇄 정책이 아니라, 브랜드 포지셔닝과 이야기거리를 만들기 위한 진입 조건으로 운용한다.

창립회원은 2026년 한정으로 큐닷의 성장과 함께 성장할 초기 네트워크로 정의한다. 지인의 명함스캔, 초대, 현장 시연, 전문가 네트워크를 통해 가입시키며, 골드번호 보유 여부와 관계없이 가입 가능하게 한다.

전문가회원은 변호사, 세무사, 보험/부동산/자동차/기업영업 등 명함과 신뢰가 중요한 직군을 중심으로 확장한다. 이들은 Q메일을 단순 이메일이 아니라 명함 신뢰, 번호 자부심, 고객 응대의 도구로 사용할 가능성이 높다.

핵심 포지셔닝:

- 당신의 골드번호, 이젠 메일도 삼켰습니다.
- 골드번호에는 골드도메인이 딱.
- 번호 자부심이 있는 사람의 Q메일.
- 명함으로 신뢰를 구축하는 사람들의 번호메일.

## 2. 회원 구분

| 구분 | 가입 조건 | 전략적 의미 |
|---|---|---|
| 일반 무료회원 | 골드번호 보유자 | 희소성, 화제성, 번호 자부심 |
| 2026 창립회원 | 지인 명함스캔, 초대, 현장 가입 | 초기 신뢰 네트워크 구축 |
| 전문가회원 | 전문직/전문영업인/명함 기반 직군 | 반복 사용, 소개 확산, 신뢰 포지션 |
| 행사 가입회원 | B2B 행사, 세미나, 네트워킹 현장 | 대량 시연, 빠른 전환 |

## 3. 가입경로 4종

### 경로 A. 지인을 만나 명함스캔

상황:
오프라인 미팅에서 상대 명함을 스캔하고, 즉시 Q메일 발급 또는 명함 저장을 진행한다.

현재 시스템 흐름:

1. 명함 촬영
2. OCR 결과 확인
3. 이름, 휴대전화번호, 생성될 Q메일, 연결메일 확인/수정
4. 전화번호와 연결메일이 있으면 Q메일 발급 또는 전달
5. 둘 중 하나가 없으면 명함만 저장 제안

성공 기준:

- 이메일/전화번호가 없어도 같은 수정 화면이 떠야 한다.
- 전화번호 입력 시 생성될 Q메일이 즉시 갱신되어야 한다.
- 연결메일이 없으면 카톡전송/QR전달 전에 차단되어야 한다.
- 발급될 Q메일이 연결메일로 잘못 들어가면 안 된다.
- 비포 명함에는 실제 명함 이메일 또는 수정 입력한 이메일이 보여야 한다.

개선 후보:

- 스캔 결과 화면에서 필수값 상태를 시각적으로 표시한다.
- "Q메일 발급"과 "명함만 저장"의 차이를 버튼명으로 더 명확히 분리한다.
- 양면 명함 스캔 플로우를 추가한다.

### 경로 B. 카톡으로 링크/QR 전송 후 홈페이지 가입신청서 유도

상황:
A가 상대방에게 카톡 링크 또는 QR을 보내고, B가 본인 휴대폰에서 가입을 완료한다.

현재 시스템 흐름:

1. A가 명함스캔 후 전달하기
2. 전화번호와 연결메일 필수 확인
3. 임시 레코드 생성
4. 카톡 링크 또는 QR 전달
5. B가 링크 진입
6. 가입신청서 확인/수정
7. 카카오 인증
8. Q메일 발급 완료

성공 기준:

- 전달하기 전에 휴대전화번호와 기존 연결메일이 모두 있어야 한다.
- B 진입 시 자동 제출되지 않고, 입력/수정 화면을 보여줘야 한다.
- B가 연결메일을 수정할 수 있어야 한다.
- 카카오 인증 후 기존 temp 레코드가 active로 승격되어야 한다.
- 완료 화면에서 Q메일과 프로필 QR이 정상 표시되어야 한다.

개선 후보:

- A가 전달 전 "상대에게 전달될 정보"를 한 번 더 볼 수 있게 한다.
- B 진입 화면에서 "이미 A가 입력한 값"과 "내가 수정할 값"을 구분한다.
- 링크 만료 또는 temp 레코드 없음 상태의 안내 화면을 만든다.

### 경로 C. 무작위 홈페이지 유입

상황:
검색, 공유, 광고, QR 등으로 사용자가 직접 홈페이지에 들어와 가입을 시도한다.

정책상 의도:
일반 무료회원은 골드번호 보유자로 한정한다. 골드번호가 아닌 사용자는 창립회원 초대, 명함스캔, 전문가회원 신청으로 우회 가입할 수 있다.

검증할 흐름:

1. 홈페이지 진입
2. 휴대전화번호 입력
3. 골드번호 여부 판정
4. 골드번호면 무료 가입 진행
5. 골드번호가 아니면 창립회원/전문가회원/초대가입 안내

성공 기준:

- 사용자가 "나는 가입 불가"라고 즉시 이탈하지 않도록 대체 경로를 보여줘야 한다.
- 골드번호 판정 기준이 명확해야 한다.
- 무료회원, 창립회원, 전문가회원의 차이가 간단히 보여야 한다.
- 골드번호가 아니어도 명함스캔/초대/전문가 신청으로 이어져야 한다.

개선 후보:

- 골드번호 판정 함수와 UI를 추가한다.
- "내 번호가 골드번호인지 확인" 기능을 첫 화면 또는 가입폼에 배치한다.
- 골드번호가 아니면 "2026 창립회원 초대받기" 또는 "전문가회원 신청" CTA를 제공한다.

### 경로 D. B2B 행사 등 참여 후 대량가입

상황:
세미나, 박람회, 협회 모임, 영업조직 설명회 등에서 다수의 명함을 스캔하거나 현장에서 가입을 유도한다.

운영 흐름:

1. 행사 전 전용 QR/초대코드 생성
2. 현장 명함스캔 또는 QR 가입
3. 참가자별 Q메일 발급
4. 행사 종료 후 가입자 명단/명함첩 정리
5. 후속 안내 메일 또는 카톡 발송

성공 기준:

- 같은 기기에서 연속 명함스캔이 빠르게 가능해야 한다.
- 이미 가입된 번호는 중복 가입으로 막아야 한다.
- 명함만 저장과 Q메일 발급이 혼동되지 않아야 한다.
- 행사별 유입 출처를 기록할 수 있어야 한다.
- 대량 가입 후 후속 관리가 가능해야 한다.

개선 후보:

- 행사 코드 또는 referrer_qid 외 event_code 컬럼/파라미터를 추가한다.
- "다음 명함 스캔" 버튼을 더 눈에 띄게 만든다.
- 행사 운영자용 간단 대시보드가 필요하다.
- 가입 완료/미완료/temp 상태를 한 화면에서 확인할 수 있어야 한다.

## 4. 시뮬레이션 방법

각 경로는 실제 운영 전 아래 순서로 반복 테스트한다.

### 4.1 테스트 데이터 세트

| 케이스 | 이름 | 전화번호 | 명함 이메일 | 기대 결과 |
---|---|---|---|---|
| S1 | 있음 | 010 있음 | 있음 | 즉시 발급 가능 |
| S2 | 있음 | 010 있음 | 없음 | 연결메일 입력 요구 |
| S3 | 있음 | 010 없음 | 있음 | 휴대폰 입력 요구 |
| S4 | 있음 | 010 없음 | 없음 | 휴대폰+연결메일 입력 요구, 명함만 저장 제안 |
| S5 | 있음 | 골드번호 | 있음 | 일반 무료회원 가능 |
| S6 | 있음 | 비골드번호 | 있음 | 창립/전문가/초대 경로 안내 |
| S7 | 이미 가입된 번호 | 010 있음 | 있음 | 중복 가입 차단 |
| S8 | 양면 명함 | 앞면/뒷면 분산 | 있음 | 양면 스캔 필요 안내 |

### 4.2 경로별 체크 항목

각 경로별로 다음 항목을 기록한다.

- 진입 경로
- 소요 시간
- 사용자가 멈춘 지점
- 필수값 누락 시 안내 문구
- Q메일 생성값
- 연결메일 저장값
- temp/active 상태 전환
- 카카오 인증 성공 여부
- 최종 프로필/QR 표시 여부
- 명함 저장 여부
- 운영자가 후속 관리 가능한지

### 4.3 성공 판정

각 경로의 성공 조건:

- 사용자가 다음 행동을 명확히 이해한다.
- 전화번호와 연결메일이 없을 때 서버 오류가 나지 않는다.
- Q메일 발급과 명함 저장이 구분된다.
- 가입 완료 후 Q메일 수신 테스트가 가능하다.
- 운영자가 가입 상태를 확인할 수 있다.

## 5. 현재 시스템 기준 우선 개선 과제

1. 골드번호 판정 로직 추가
   - 뒤 4자리 기준으로 `AAAA`, `A000`, `000A`, `AA00`, `00AA`, `ABAB`, `AABB`, `ABCD`, `DCBA`, 의미번호를 분류한다.

2. 가입 경로 태깅
   - scan_contact, kakao_link, homepage, event 같은 source 값을 temp/registration에 남긴다.

3. 창립회원/전문가회원 플래그
   - founder_2026, professional, gold_number 같은 회원 속성을 분리한다.

4. B2B 행사 코드
   - event_code 또는 campaign_code를 링크에 붙여 행사별 유입을 구분한다.

5. 관리자 확인 화면
   - temp, active, card_only, duplicate 상태를 한 화면에서 확인할 수 있게 한다.

6. 양면 명함 스캔
   - "뒷면 추가 스캐" 또는 "추가 이미지 업로드" 버튼을 OCR 결과 화면에 붙인다.

## 6. 운영 메시지 초안

일반 무료회원:

> 010.Q 무료회원은 골드번호 보유자에게 우선 제공됩니다.
> 당신의 골드번호, 이제 메일까지 삼킵니다.

창립회원:

> 2026년 창립회원은 지인의 명함스캔 또는 초대를 통해 가입할 수 있습니다.
> 큐닷의 성장과 함께 성장할 초기 멤버를 모십니다.

전문가회원:

> 명함으로 신뢰를 만드는 분들을 위한 번호메일.
> 변호사, 세무사, 전문영업인, 컨설턴트에게 Q메일은 새로운 디지털 명함입니다.

행사 가입:

> 오늘 만난 명함, 오늘 바로 Q메일로 연결됩니다.
> 행사장에서 즉시 발급받고 메일 수신까지 확인하세요.

## 7. 결론

2026년 회원확보 전략은 대량 무작위 가입보다 골드번호, 창립회원, 전문가회원, 행사 기반 가입의 네 축으로 설계한다.

골드번호는 가입 제한 장치이면서 동시에 강한 이야기거리다. 창립회원과 전문가회원 경로는 실제 가입 제한을 완화하면서도 브랜드의 희소성과 신뢰성을 유지하는 통로가 된다.

따라서 시스템 개선은 단순 가입폼 개선보다 다음 네 가지에 집중한다.

1. 골드번호 판정
2. 가입경로 태깅
3. 명함스캔 기반 창립회원 흐름 안정화
4. 행사/전문가 대량가입 운영성 확보
