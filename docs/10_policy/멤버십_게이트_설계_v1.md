# 멤버십 게이트(가입 자격·경로 통제) 설계 v1.1
_작성: 2026-07-01 · v1.1: 대표 결정 5건 확정 반영_

> 핵심: "아무나 가입 못 하게 막고, 정해진 자격·경로로만 가입시키는 입구 로직."
> 현재 가입 플로우 위에 **판정(허용/부여등급/안내)** 레이어를 얹는다.

## 0. 확정된 정책 (대표 결정)

| 항목 | 확정값 |
|---|---|
| **골드번호 정의** | **통신사 골드번호 추첨 기준 패턴**(뒷자리 9종) + 특수의미는 수동 테이블 (§3) |
| **전문직 자격** | 정부 라이선스 직업(의사·변호사·공인중개사 등). 자격증 제출 증빙. **단, 2026년은 "명함"으로 갈음** |

| **위반 시 동작** | **차단·강등 없음** → 계정 미생성 + 정보성 안내(올바른 경로 재시도 가능) |
| **Founder 모집 마감** | **2026-12-31** (`FOUNDER_DEADLINE`) |

## 2. 등급 & 추천 권한 매트릭스

**추천인이 가입시킬 수 있는 대상:**

| 추천인 | 골드번호 | 일반번호 | 비고 |
|---|---|---|---|
| **Founder** | ✅ | ✅ (명함스캔) | 전권 |
| **골드번호 회원** | ✅ | ❌ | **골드만** |
| **일반 회원** | ❌ | ❌ | 추천 권한 없음 |
| **추천 없음(직접)** | 골드 보유자 본인만 ✅ | Phase 2에서만 | 시기 의존 |

**자가가입:** 골드 보유자 본인 = 직접 허용 / 전문직 = 명함(2026)·자격증(2027~) 증빙 시 허용 / 일반번호 = 창립기간엔 Founder 명함스캔만, Phase 2에 홈페이지 직접.
**Founder 등급:** `Q.지정처` 또는 `Founder 추천` 경로로만.

## 3. 골드번호 판별 (통신사 추첨 기준 기반)

**자동 패턴(뒷 8자리 = 국번4 `MMMM` + 가입번호4 `NNNN`):**

| 유형 | 조건 | 예 |
|---|---|---|
| AAAA | `NNNN` 네 자리 동일 | 7777 |
| ABAB | `NNNN` = a b a b | 1212 |
| ABCD 연속 | `NNNN` 연속 증가(또는 감소) | 1234 / 4321 |
| ABCD-ABCD 대칭 | `MMMM == NNNN` (국번=가입번호) | 1234-1234 |
| A000 | a≠0, 나머지 000 | 5000 |
| 000A | 앞 000, 끝 a≠0 | 0007 |
| AA00 | a a 0 0 | 7700 |
| 00AA | 0 0 a a | 0077 |
| **특수의미** | 자동판별 불가 | **`gold_numbers` 수동 테이블** |

```
function isGoldNumber(phone) {
  const d = phone.replace(/\D/g,'');            // 01012345678
  if (d.length !== 11) return false;
  const mid = d.slice(3,7), last = d.slice(7);  // MMMM, NNNN
  const [a,b,c,e] = last.split('');
  if (a===b && b===c && c===e) return true;                 // AAAA
  if (a===c && b===e) return true;                          // ABAB
  const asc  = (+b===+a+1 && +c===+b+1 && +e===+c+1);
  const desc = (+b===+a-1 && +c===+b-1 && +e===+c-1);
  if (asc || desc) return true;                             // ABCD 연속
  if (mid === last) return true;                            // ABCD-ABCD
  if (a!=='0'&&b==='0'&&c==='0'&&e==='0') return true;      // A000
  if (a==='0'&&b==='0'&&c==='0'&&e!=='0') return true;      // 000A
  if (a===b&&c==='0'&&e==='0'&&a!=='0') return true;        // AA00
  if (a==='0'&&b==='0'&&c===e&&c!=='0') return true;        // 00AA
  return goldTableHas(d);                                   // 특수의미(수동)
}
```
> 통신사별 세부 기준이 갱신되면 패턴 표만 조정. 근거: SKT/KT 골드번호 추첨 유형.

**수동 테이블(특수의미·예외):**
```
gold_numbers(number text PK, tier text, note text, added_at timestamptz)
```

## 4. 판정 알고리즘

```
function gateCheck({ phone, referrer_qid, path, proAttested, now }) {
  const isGold = isGoldNumber(phone);
  const ref    = referrer_qid ? lookupGrade(referrer_qid) : null;   // founder|gold|general|pro|null
  const founderWindow = now <= '2026-12-31';

  // 1) 골드번호
  if (isGold) {
    if (ref?.grade==='founder' || ref?.grade==='gold' || selfOwnsGold(phone))
      return ALLOW('gold');
    return REJECT('GOLD_NEEDS_FOUNDER_OR_GOLD');   // 차단·강등 아님: 계정 미생성 + 안내
  }
  // 2) 전문직 (명함 2026 / 자격증 2027~)
  if (proAttested) return ALLOW('pro');

  // 3) 일반번호
  if (founderWindow) {
    if (ref?.grade==='founder' && path==='card_scan') return ALLOW('general');
    return REJECT('GENERAL_NEEDS_FOUNDER_INVITE');
  }
  return ALLOW('general');   // Phase 2(일반오픈)
}
```
**REJECT = 차단/강등이 아니라 "이 경로로는 가입 불가" 안내.** 밴 기록·강등 계정 생성 없음. 올바른 경로로 언제든 재시도 가능.

## 5. 데이터 모델 추가

- `registrations.grade text` — 'founder'|'gold'|'pro'|'general' (자격축; 상품축 `plan`과 분리)
- `gold_numbers` 테이블 (특수의미·예외 수동 관리)
- 전문직 증빙: `pro_attest` (2026=명함 이미지 경로, 2027~=자격증) — 검증 상태 컬럼
- 서버 상수: `FOUNDER_DEADLINE='2026-12-31'`, 게이트 on/off 플래그(config)

---

## 6. 구현 위치

- **서버**: `functions/api/register.js` — INSERT 직전 `gateCheck()`. REJECT면 `{ ok:false, code, reason }`(DB 미기록). ALLOW면 `grade` 기록.
- **프론트**: 사유별 안내
  - `GOLD_NEEDS_FOUNDER_OR_GOLD` → "골드번호는 골드회원·파운더 초대로만 가입돼요"
  - `GENERAL_NEEDS_FOUNDER_INVITE` → "지금은 파운더 초대(명함스캔)로만 가입할 수 있어요"
- **스위치**: `FOUNDER_DEADLINE` 경과 시 Phase 2 자동 전환(코드 배포 불필요)

## 8. 구현 규모 & 착수

- 모든 결정 **확정 완료** → 바로 구현 가능
- 작업: `isGoldNumber()` + `gateCheck()` + `register.js` 연결 + `grade`/`gold_numbers`/`pro_attest` DB + 프론트 안내 → **약 1일 규모**
- **지금**: 창립 모집은 초대(파운더 스캔) 기반이라 **수동 게이트로 시작 가능**. 자동 게이트는 일반오픈(2027) 전 구현이면 충분.
