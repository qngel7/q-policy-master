# Q-ID 이름이니셜 정정 정책 v1.0

> 확정일: 2026-07-22  
> 적용 서비스: Q. 통합계정·010Q  
> 상태: 대표님 승인 완료

## 1. Q-ID의 역할

Q-ID는 회원이 보고 기억하고 공유하는 공개 신원이다.

```text
Q-ID = 이름이니셜 영문 3자리 + 전 회원 고유순번 8자리
예: PMS00000016
```

`account_id`는 로그인·패스키·프로필·활동기록을 같은 회원에게 연결하는 시스템 내부 불변값이다. 회원에게 표시하거나 기억하도록 요구하지 않는다.

## 2. 변경 가능 범위

| 구성 | 예시 | 변경 여부 |
|---|---|---|
| 이름이니셜 3자리 | `PMS` | 로그인한 본인이 자신의 이름이니셜 범위에서 수정 가능 |
| 고유순번 8자리 | `00000016` | 평생 변경 불가 |
| 내부 `account_id` | UUID | 시스템 내부에서 영구 불변 |

고유순번은 전 회원에게 하나씩만 발급되므로 이름이니셜을 수정해도 다른 회원과 Q-ID가 중복되지 않는다.

## 3. 자동 발급 원칙

- 신규 가입의 기본 이니셜은 서버가 한글 이름의 음절별 로마자 첫 글자로 계산한다.
- 가입 화면이 전송한 임시 Q-ID 접두어를 서버가 그대로 신뢰하지 않는다.
- `PARKMYOUNGSOON`처럼 공백 없는 영문명 한 단어의 앞 세 글자 `PAR`을 이름이니셜로 사용하지 않는다.
- 로마자 표기는 개인차가 있으므로 자동 계산값은 추천값이며, 가입 후 본인이 영문 3자리로 바로잡을 수 있다.

## 4. 정정 원칙

- 본인 Q. 통합계정 로그인 상태에서만 정정한다.
- 영문 대문자 3자리만 허용한다.
- 정정 시 기존 8자리 고유순번과 `account_id`를 유지한다.
- 로그인·패스키·프로필·Q메일·활동 및 멤버십 기록을 그대로 승계한다.
- 과거 Q-ID는 변경이력으로 보관하고 현재 Q-ID 프로필로 연결한다.
- 기존 추천관계와 저장 명함의 소유자 Q-ID를 현재 Q-ID로 갱신한다.
- 변경 전에 만들어진 과거 Q-ID 초대 링크도 현재 추천인 Q-ID로 해석한다.

## 5. 최초 적용 사례

16번 회원 박명순의 Q-ID는 공백 없는 영문명 처리 오류로 `PAR00000016`이 발급되었다. 이름이니셜 원칙에 따라 고유순번을 유지하고 다음과 같이 정정한다.

```text
PAR00000016 → PMS00000016
```

서대원 추천관계와 회원의 모든 계정·활동 정보는 그대로 유지한다.
