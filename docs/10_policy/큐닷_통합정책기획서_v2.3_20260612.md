# §3 010Q · OKQ 메일 시스템 (v2.3 개정)

> **핵심 명제:** 번호가 곧 이메일, 이메일이 곧 번호 — 무료(010Q)와 유료(OKQ)가 명확히 분리된다.

## §3-A 010Q메일 (무료 · 기본 등급)

| 항목 | 내용 |
|---|---|
| **가입 경로** | 홈페이지 접속 (추천링크 또는 QR코드) |
| **가입 조건** | 휴대폰 본인인증 |
| **발급 성격** | **인증 즉시 발급** — 선점 예약이 아님, 실사용 주소 즉시 활성 |
| **가격** | 무료 · 영구 |
| **이메일 형식** | `01070008000@Q.co.kr` (010 포함 11자리) |
| **타깃** | 전 국민 (휴대폰 보유자) |
| **Q-vic 적립** | ✅ 활동(출석·설문·추천 등)으로 획득 가능 |
| **Q-vic 사용** | ❌ 이니셜 구독(OKQ 플랜) 가입 후 사용 가능 |

### 가입 흐름

```
추천링크·QR 접속
    ↓
홈페이지 신청서 작성 (번호 입력)
    ↓
휴대폰 본인인증 (카카오 OAuth 또는 SMS)
    ↓
010Q 메일 + Q-ID 즉시 발급 ← 선점·예약 없음, 실사용 주소 바로 활성
```

## §3-B OKQ메일 (유료 · 이니셜 구독 등급)

### OKQ 네이밍 의도 (v2.3 신설)

> **OKQ = "010, OK, Q"**
> - **010 붙여도·빼도 수신 OK** (7000-8000@Q.co.kr 과 010-7000-8000@Q.co.kr 모두 수신)
> - 명함에 번호 없이 이름만 쓰는 시대 — 이름@Q.co.kr 로 수신
> - **명함 사용자(직장인·전문가)** 특화 서비스

| 항목 | 내용 |
|---|---|
| **가입 경로** | 명함 스캔 (기본) 또는 010Q에서 구독 업그레이드 |
| **가격** | **월 ₩1,500 (= $1) · 이니셜 구독** |
| **이메일 형식** | `70008000@Q.co.kr` (010 제외 8자리) |
| **수신 호환** | `01070008000`, `010-7000-8000`, `70008000`, `7000-8000` 모두 수신 |
| **내이름메일** | `김한국@Q.co.kr` — 한글 이름 기반 자동 예약 (양도 가능) |
| **영문여권이름메일** | `KIMHANKOOK@Q.co.kr` — 여권 기준 대문자 추가 생성 (양도 가능) |
| **타깃** | 직장인·전문가·명함 보유자 |
| **Q-vic 사용** | ✅ 취득 및 자유 사용 |
| **Q-bit 사용** | ✅ 취득 및 Q-bit 전환 |

### 명함 없는 사용자의 OKQ 진입 경로 (v2.2 유지)

- 명함 미보유자는 다이어트 명함 앱이 AI로 임시 명함 자동 생성 → OCR 입력 명함으로 OKQ 가입 조건 충족
- 리멤버·토스명함 등 기존 명함 앱을 통해서도 가입 가능 (API 연동 협의 중)
- 또는 010Q 플랜 이용 중 언제든 이니셜 구독 계약으로 업그레이드

## §3-C 이메일 부가 주소 정책 (v2.3 신설)

| 주소 유형 | 형식 예시 | 발급 조건 | 양도 |
|---|---|---|---|
| 010Q 기본 | 01070008000@Q.co.kr | 010Q 가입 즉시 | ❌ |
| OKQ 기본 | 70008000@Q.co.kr | OKQ 구독 유지 중 | ❌ |
| 내이름메일 (한글) | 김한국@Q.co.kr | OKQ 구독 시 자동 예약 | ✅ (1회) |
| 영문여권이름메일 | KIMHANKOOK@Q.co.kr | OKQ 구독 시 추가 생성 | ✅ (1회) |

**양도 정책:** 양도되는 것은 이름 자산(한글이름메일, 영문여권이름메일 등 주소 자체)에 한함. Q-vic·수발신 이력은 기존 소유자의 Q-ID에 그대로 보존되며 새 소유자에게 이전되지 않는다. 모든 거래 내역은 Q-ID에 귀속되므로 주소 양도 후에도 원소유자의 기록은 유지된다.

## §6-3 Q-vic 획득 활동 (v2.3 개정)

> **핵심 원칙:** Q-vic이 아무리 많아도 사용하려면 이니셜 구독(OKQ 이상)을 해야 한다. Q-vic으로 이니셜 구독료 결제는 불가. 단, 서브도메인 구매·플랫폼 내 사용료 결제는 Q-vic으로 가능.

| 활동 | Q-vic 획득 | 어뷰징 방지 | 비고 |
|---|---|---|---|
| 출석 | 미정 (v3.0 리서치 후 확정) | 1일 1회 | — |
| 검색 | 미정 (v3.0 리서치 후 확정) | 체류 조건 | — |
| 설문 | 미정 (v3.0 리서치 후 확정) | AI 품질 검증 | — |
| 회원 추천 | 미정 (v3.0 리서치 후 확정) | 가입+유지 조건 | — |

**Q-vic 사용 가능 항목 (이니셜 구독 활성 시)**

- 서브도메인 구매·임대료
- 명함 주문
- 플랫폼 내 프리미엄 기능
- 생태계 내 결제 전반

**Q-vic 사용 불가 항목**

- 이니셜 구독료(₩1,500/월) — 현금·카드 결제만 가능

# §10 Q-vic · Q-bit 경제 시스템 (v2.2 유지 + 가격 반영)

## §10-1 이니셜 구독과 Q-vic 관계 원칙

- **1 Q-vic = ₩1** (유지)
- **이니셜 구독료(₩1,500/월)는 Q-vic으로 결제 불가** — 현금·카드만 허용
- 이니셜 구독을 납부하면 Q-vic을 **생태계 내 어디서나** 자유롭게 사용 가능
- 이니셜 구독 미납 시 Q-vic 잔액은 동결 (적립은 계속)

## §10-2 Q-vic 흐름도 (v3.0 리서치 후 가격 재반영 예정)

> 활동별 Q-vic 지급 단가는 v3.0에서 리서치 후 확정. 아래는 구조 원칙만 명시.

```
[유입: Q-vic 적립]                    [유출: Q-vic 사용]
┌─────────────────┐                  ┌─────────────────┐
│ 출석             │                  │ 서브도메인 구매  │
│ 검색             │ ──── Q-vic ────> │ 명함 주문        │
│ 설문·추천        │   (단가 v3.0)    │ 프리미엄 기능    │
└─────────────────┘                  │ 생태계 결제 전반 │
                                     └─────────────────┘
  ※ 이니셜 구독료 결제는 Q-vic 사용 불가
  ※ 이니셜 구독 납부 시 → Q-vic 생태계 전체 오픈
     ↕
  [ Q-bit 전환 (OKQ 이상, 이니셜 활성 시) ]
```
